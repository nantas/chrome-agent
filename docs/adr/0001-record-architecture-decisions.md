# 0001 — 记录架构决策

## 状态

已接受

## 背景

项目需要一个一致的方式来记录重要架构决策，以便团队成员和 agent 理解设计选择背后的原因。

## 决策

采用 Architecture Decision Records (ADR) 模式，将决策记录在 `docs/adr/` 目录下。每份 ADR 包含：

- **状态**：已提出 / 已接受 / 已弃用 / 已取代
- **背景**：决策的上下文和驱动力
- **决策**：做出的具体选择
- **后果**：决策带来的影响（正面和负面）

命名格式：`NNNN-简短标题.md`（四位数编号，kebab-case 标题）。

## 后果

- 所有重大架构变更都有可追溯的记录
- Agent skill（如 `grill-with-docs`）可以引用 ADR 进行设计讨论
- 需要维护 ADR 的编号连续性和内容准确性
