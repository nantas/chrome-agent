# 认证会话规则

## 基本规则

- 已登录/认证的工作需要用户明确批准目标页面或标签页后才能尝试
- 认证运行默认为只读，除非用户明确扩大范围（如写操作）
- Scrapling-first 仍适用于已批准的认证工作

## 会话复用

### Scrapling 会话路径

1. 尝试使用 Scrapling session 变体复用认证状态
2. 如果成功 → 用 Scrapling 完成抓取
3. 如果失败（重定向到登录、页面重置、上下文丢失）→ 停止 Scrapling 路径

### 实时标签页 Fallback

如果存在已批准的实时标签页：
1. 切换到 repo-local chrome-cdp
2. 在实时标签页上继续操作
3. 保持只读，除非用户明确扩大范围

## 安全停止条件

遇到以下任一情况时，**停止并记录失败**而非尝试继续：
- 重定向到登录页面
- 页面被重置或登出
- 出现写操作风险（意外 POST、DELETE 等）
- 已批准的上下文丢失

## 已验证案例

### x.com 搜索页
- Scrapling-first 仍为开场动作
- 但某些站点（如 x.com）在 session 复用失败后可能需要立即 fallback 到 chrome-cdp
- 当前会话延续优先级高于强推 Scrapling
